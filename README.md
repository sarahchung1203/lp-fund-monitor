## 한눈에 보는 기능

| 기능 | 설명 |

## 동작 구조

```
매일 08:30 KST (GitHub Actions cron, 클라우드 실행 — PC 불필요)
   │
   ├─ scrapers.py   : 10개 소스 수집 (기관별 파서 + 펀드 공고 선별 필터)
   ├─ build.py      : 무관공고 제외 → 소스간 중복 제거 → 첨부 PDF에서 마감일 추출(캐시)
   │                  → NEW/마감 판별(KST) → index.html + data.json 생성
   └─ git push      : GitHub Pages가 자동 재배포 → 대시보드 URL 갱신
```

### 수집 견고성 — 3중 안전망

수집이 통째로 실패하지 않도록 모든 소스에 다음이 적용되어 있습니다.

1. **HTTP 자동 재시도**: 타임아웃/일시 오류 시 백오프를 두고 최대 4회 재시도 (+소스 단위 3회).
2. **Jina Reader 폴백**: 일부 한국 사이트(예: 군인공제회)는 해외 클라우드 IP를 차단합니다. 직접 접속이 막히면 `r.jina.ai` 프록시로 원문 HTML/JSON을 우회 수집합니다 (`scrapers._get_kr` / `_get_json_kr`).
3. **직전 수집분 이월(carry-forward)**: 그래도 실패하면 해당 소스의 마지막 성공 데이터를 유지하고 "일시적 수집 실패 — 직전 수집분 표시 중" 안내만 띄웁니다. **소스 하나가 죽어도 대시보드가 비지 않습니다.**

### 마감일 자동 추출 커버리지

| 소스 | 마감일 출처 |
|------|------------|
| KVCA | 목록의 마감일 컬럼 |
| KFCC | 상세 본문의 `접수기한` 텍스트 |
| 한국성장금융 | **첨부 공고 PDF의 선정일정 표**에서 `제안서 접수` 행 날짜 추출 (결과는 `state.json`의 `deadline_cache`에 저장, 매일 신규 항목만 다운로드) |
| KVIC·신한벤처투자 | 공고 PDF 폰트가 깨져(HWP→PDF 변환) 숫자 추출 불가 → **오탐 방지를 위해 비워 둠** (OCR 필요) |

## 저장소 구성

| 파일 | 역할 |
|------|------|
| `scrapers.py` | 기관별 수집 로직 + 펀드 공고 선별 필터 + 첨부 PDF 마감일 추출 |
| `build.py` | 파이프라인 (필터→중복제거→NEW/마감 판별→HTML 생성) |
| `index.html` | **대시보드** (데이터 내장 단일 파일 — 이 파일만 열어도 동작) |
| `data.json` | 최신 수집 결과 (기계가 읽는 용도) |
| `state.json` | 공고별 최초 발견일 + 마감일 추출 캐시 |
| `.github/workflows/daily.yml` | 매일 08:30 KST 자동 실행 (GitHub Actions) |
| `requirements.txt` | Python 의존성 (`requests`, `beautifulsoup4`, `lxml`, `pymupdf`) |
| `run.bat` | (선택) 내 PC에서 즉시 수동 갱신 — pull → 수집 → push |

## 사용 방법

- **그냥 보기**: 위 대시보드 URL을 즐겨찾기 (휴대폰 홈 화면에 추가하면 앱처럼 사용 가능)
- **지금 즉시 갱신**: GitHub 저장소 → **Actions** 탭 → "Daily LP notice scrape" → **Run workflow** 버튼 (또는 내 PC에서 `run.bat` 더블클릭)
- **자동 실행 시간 변경**: `.github/workflows/daily.yml`의 cron 수정 (UTC 기준 — `30 23 * * *` = 08:30 KST)

## 직접 설치하기 (다른 팀/계정에서 쓰려면)

이 저장소만 있으면 서버 없이 무료로 동일한 시스템을 돌릴 수 있습니다.

1. 이 저장소를 **Fork** (또는 코드 복사 후 새 public 저장소에 push)
2. 저장소 **Settings → Pages** → Source: `main` 브랜치 `/` (root) 선택
3. **Actions** 탭에서 워크플로 활성화 (Fork 직후 한 번 승인 필요)
4. 끝 — 매일 자동 수집되며, 대시보드는 `https://<계정명>.github.io/<저장소명>/`

로컬 실행만 원하면: `pip install -r requirements.txt` 후 `python build.py` → 생성된 `index.html`을 브라우저로 열기.

## 유지보수 노트

- **NEW 기준 일수**: `build.py` 상단 `NEW_DAYS` (기본 3일, 게시일 기준)
- **수집 페이지 수**: 각 `scrape_*` 함수의 `pages` 인자 (예: 교직원공제회는 펀드 공고가 뒤 페이지에 밀려 10페이지 스캔)
- **특정 기관이 계속 0건이면**: 사이트 구조가 바뀐 것 → 해당 `scrape_*`의 CSS 선택자 점검
- **마감일을 다시 추출하려면**: `state.json`의 `deadline_cache`에서 해당 항목(또는 전체) 삭제 후 재실행
- **필터 조정**: 무관 공고 제외는 `build.py`의 `GLOBAL_EXCLUDE`, 펀드 공고 판별은 `scrapers.py`의 `_is_fund_sel` / `_is_kofia_fund`
