# LP 출자사업 공고 모니터링

5개 기관의 출자사업/공고 페이지를 매일 자동으로 수집해 **하나의 대시보드(index.html)**로 모아 보는 시스템입니다.
제목을 클릭하면 원문 공고 페이지로 바로 이동하며, 최근에 새로 올라온 공고는 **NEW** 배지로 표시됩니다.

## 수집 대상

| 기관 | 페이지 | 비고 |
|------|--------|------|
| 한국벤처투자(모태펀드) | KVIC 출자사업 공고 | 전체 |
| 한국성장금융 | K-Growth 공지사항 | 전체 |
| 벤처캐피탈협회 | KVCA 출자공고 | 전체 (주관기관·마감일 포함) |
| 새마을금고중앙회 | KFCC 공지사항 | **[펀드] / [운용사] 포함 공고만** |
| 신한벤처투자 | Shinhan 공지사항 | 전체 |

## 구성 파일

| 파일 | 설명 |
|------|------|
| `scrapers.py` | 기관별 수집 로직 |
| `build.py` | 수집 → 신규(NEW) 판별 → `index.html` / `data.json` 생성 |
| `index.html` | **대시보드** (데이터 내장, 이 파일만 열면 됨) |
| `data.json` | 최신 수집 결과(JSON) |
| `state.json` | 공고별 최초 발견일 기록 (NEW 판별용) |
| `run.bat` | 일일 자동 실행 스크립트 (작업 스케줄러용) |

## 사용 방법

- **그냥 보기**: `index.html` 더블클릭 → 브라우저에서 열림
- **수동 갱신**: `run.bat` 더블클릭 (또는 `python build.py`)
- **자동 갱신**: Windows 작업 스케줄러에 `LP출자공고모니터링` 작업이 등록되어 **매일 오전 8:30** 자동 실행됩니다.

### 자동 실행 시간 변경
작업 스케줄러(`taskschd.msc`) → `LP출자공고모니터링` → 트리거 편집,
또는 PowerShell:
```powershell
Set-ScheduledTask -TaskName "LP출자공고모니터링" `
  -Trigger (New-ScheduledTaskTrigger -Daily -At 9:00AM)
```

## 동작 메모

- **NEW 배지**: 공고가 처음 수집된 날로부터 `NEW_DAYS`(기본 3일) 동안 표시됩니다. (`build.py` 상단에서 조정)
- 수집 페이지 수는 각 `scrape_*` 함수의 `pages` 인자로 조정합니다. (KFCC는 [펀드]/[운용사] 공고가 드물어 기본 6페이지 스캔)
- 한 기관 수집이 실패해도 나머지는 정상 동작하며, 대시보드 상단에 실패 안내가 표시됩니다.
- 사이트 구조가 바뀌어 특정 기관 건수가 0이 되면 해당 `scrape_*` 함수의 CSS 선택자를 점검하세요.
