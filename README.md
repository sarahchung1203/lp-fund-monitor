# LP 출자사업 공고 모니터링

10개 기관/집약 게시판의 출자사업 공고를 매일 자동 수집해 **하나의 대시보드(index.html)**로 모아 보는 시스템입니다.
제목을 클릭하면 원문 공고 페이지로 바로 이동하며, 최근에 새로 올라온 공고는 **NEW** 배지로 표시됩니다.

### 필터링 / 중복 제거
- **펀드 출자 공고만**: 채용·포럼·시스템공지 등 무관 공고는 전 소스 공통 제외(`GLOBAL_EXCLUDE`). KOFIA는 공개시장 운용 공고까지 추가 제외(`scrapers._is_kofia_fund`).
- **소스 간 중복 제거**(`build._dedup`): 같은 펀드의 동일 종류 공고가 여러 게시판에 올라오면 하나만 남김(마감일 보유 > 소스 우선순위). 같은 펀드라도 공고/접수현황/결과/서식은 별도 유지.
- **LP 커버리지**: KVCA·KOFIA가 다수 LP(공제회·연금·중앙회 등)의 출자공고를 집약. 국민연금·산업은행 등 자체 사이트에만 게시하는 일부 기관은 개별 추가 필요.

## 구성 파일

| 파일 | 설명 |
|------|------|
| `scrapers.py` | 기관별 수집 로직 |
| `build.py` | 수집 → 신규(NEW) 판별 → `index.html` / `data.json` 생성 |
| `index.html` | **대시보드** (데이터 내장, 이 파일만 열면 됨) |
| `data.json` | 최신 수집 결과(JSON) |
| `state.json` | 공고별 최초 발견일 기록 (NEW 판별용) |
| `run.bat` | 일일 자동 실행 스크립트 (작업 스케줄러용) |

## 사용 방법

- **그냥 보기**: `index.html` 더블클릭 → 브라우저에서 열림
- **수동 갱신**: `run.bat` 더블클릭 (또는 `python build.py`)
- **자동 갱신**: Windows 작업 스케줄러에 `LP출자공고모니터링` 작업이 등록되어 **매일 오전 8:30** 자동 실행됩니다.

### 자동 실행 시간 변경
작업 스케줄러(`taskschd.msc`) → `LP출자공고모니터링` → 트리거 편집,
또는 PowerShell:
```powershell
Set-ScheduledTask -TaskName "LP출자공고모니터링" `
  -Trigger (New-ScheduledTaskTrigger -Daily -At 9:00AM)
```

## 동작 메모

- **마감 처리(그날 KST 날짜 기준)**: 마감일이 지난 공고는 박스를 흐리게(opacity) 표시하고 `마감` 태그를 붙입니다. 상단 `마감 제외` 체크 시 숨김.
  - 마감일 출처:
    - **KVCA**: 목록의 마감일 컬럼.
    - **KFCC**: 상세 본문 `접수기한` 텍스트.
    - **한국성장금융**: 첨부 공고 PDF의 **선정일정 표**에서 `제안서 접수` 행 날짜를 추출(`scrapers.attachment_deadline`). 일정표 안에서만 찾고 공고일·기준일은 제외해 오탐 방지. 추출 결과는 `state.json`의 `deadline_cache`에 저장(매일 신규 항목만 다운로드).
    - **KVIC·신한벤처투자**: 공고 PDF의 **숫자가 깨진 폰트**(HWP→PDF 변환)로 저장돼 날짜 텍스트 추출 불가 → 마감 표시 안 함(OCR 필요, 미설치). 잘못된 마감 표시를 막기 위해 비워 둠.
  - `deadline_cache`를 비우면(또는 해당 uid 삭제) 다음 실행 때 다시 추출합니다.
- **시간대**: `build.py`는 한국시간(KST)으로 '오늘'을 계산합니다. (클라우드는 UTC라 보정)
- **NEW 배지**: 공고가 처음 수집된 날로부터 `NEW_DAYS`(기본 3일) 동안 표시됩니다. (`build.py` 상단에서 조정)
- 수집 페이지 수는 각 `scrape_*` 함수의 `pages` 인자로 조정합니다. (KFCC는 [펀드]/[운용사] 공고가 드물어 기본 6페이지 스캔)
- 한 기관 수집이 실패해도 나머지는 정상 동작하며, 대시보드 상단에 실패 안내가 표시됩니다.
- 사이트 구조가 바뀌어 특정 기관 건수가 0이 되면 해당 `scrape_*` 함수의 CSS 선택자를 점검하세요.
